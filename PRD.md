# 基于 Miloco + OpenClaw 的课堂行为理解系统工作流程设计文档

## 一、项目目标

构建一个基于 **Miloco 摄像头视频流服务** 与 **OpenClaw 智能任务编排框架** 的课堂行为理解系统，实现以下功能：

1. 接入教室摄像头视频流；
2. 自动识别课堂中学生行为；
3. 统计课堂状态指标；
4. 自动生成课堂行为分析报告。

系统最终实现：

> **摄像头感知课堂画面 → 视觉大模型识别行为 → OpenClaw统计分析 → 自动生成课堂报告**

---

# 二、系统总体架构

系统整体架构如下：

```text
米家摄像头
   ↓
Miloco（生成RTSP视频流）
   ↓
OpenClaw（读取视频流并截图）
   ↓
视觉大模型（识别学生行为）
   ↓
OpenClaw（统计分析 + 报告生成）
   ↓
输出课堂行为分析报告
```

系统分为四个核心模块：

1. **Miloco视频流服务模块**
2. **OpenClaw视频处理模块**
3. **视觉大模型识别模块**
4. **课堂分析报告生成模块**

---

# 三、模块工作流程说明

---

## 3.1 Miloco视频流服务模块

### 模块目标

将米家摄像头视频转换为标准 RTSP 视频流，供 OpenClaw 调用。

---

### 工作流程

Miloco部署完成后：

1. 接入米家摄像头；
2. 获取摄像头实时画面；
3. 开启RTSP服务；
4. 生成视频流地址。

例如：

```bash
rtsp://192.168.0.136:8554/1027116728
```

其中：

* `192.168.0.136`：Miloco服务所在主机IP
* `8554`：RTSP服务端口
* `1027116728`：摄像头设备ID

---

### 模块输出

输出可供访问的视频流地址：

```bash
rtsp://<server_ip>:<port>/<camera_did>
```

---

### 模块职责总结

Miloco仅负责：

> **摄像头接入 + 视频流生成**

不负责行为识别。

---

## 3.2 OpenClaw视频处理模块

### 模块目标

通过 RTSP 地址读取摄像头视频流，并进行截图抽帧。

---

### 工作流程

OpenClaw接收RTSP流地址：

```bash
rtsp://192.168.0.136:8554/1027116728
```

然后执行：

1. 建立 RTSP 连接；
2. 拉取实时视频流；
3. 按时间间隔抽取图像帧；
4. 获取当前课堂截图。

例如：

```text
RTSP视频流 → image.jpg
```

---

### 模块输出

输出当前视频帧图片：

```bash
image.jpg
```

---

### 模块职责总结

OpenClaw在该阶段负责：

> **视频流读取 + 图像抽帧**

---

## 3.3 视觉大模型识别模块

### 模块目标

对课堂截图进行分析，识别学生行为。

---

### 行为识别内容

系统识别以下行为：

1. 举手人数
2. 睡觉人数
3. 玩手机人数
4. 记笔记人数

---

### OpenClaw调用视觉模型

OpenClaw将截图与提示词一起发送给视觉大模型：

```text
请分析当前课堂画面中学生行为：
1. 举手人数
2. 睡觉人数
3. 玩手机人数
4. 记笔记人数
请以JSON格式返回结果。
```

---

### 视觉模型返回结果示例

```json
{
  "hand_raising": 2,
  "sleeping": 1,
  "phone_usage": 3,
  "note_taking": 20,
  "total_students": 26
}
```

---

### 可选视觉模型

可使用：

* Qwen2.5-VL
* Qwen3-VL
* GPT-4V
* InternVL

---

### 模块职责总结

视觉模型负责：

> **课堂行为识别与结构化结果输出**

---

## 3.4 OpenClaw课堂状态分析模块

### 模块目标

根据视觉模型返回结果计算课堂状态指标。

---

### 统计指标设计

---

### 1. 专注率

定义：

```text
专注率 = 记笔记人数 / 总人数
```

例如：

```text
20 / 26 = 76.9%
```

---

### 2. 分心率

定义：

```text
分心率 = (睡觉人数 + 玩手机人数) / 总人数
```

例如：

```text
(1 + 3) / 26 = 15.4%
```

---

### 3. 活跃度

定义：

```text
活跃度 = 举手人数 / 总人数
```

例如：

```text
2 / 26 = 7.7%
```

---

### 统计结果示例

```json
{
  "focus_rate": 76.9,
  "distraction_rate": 15.4,
  "activity_rate": 7.7
}
```

---

### 模块职责总结

OpenClaw在该阶段负责：

> **课堂状态统计计算**

---

## 3.5 OpenClaw课堂报告生成模块

### 模块目标

根据统计指标生成课堂行为分析报告。

---

### 输入数据示例

```json
{
  "focus_rate": 76.9,
  "distraction_rate": 15.4,
  "activity_rate": 7.7
}
```

---

### 输出报告示例

```text
本节课整体专注率较高，大部分学生保持记笔记状态，
但仍有少量学生存在玩手机和睡觉现象，
课堂互动活跃度较低。
```

---

### 模块职责总结

OpenClaw负责：

> **将统计数据转换为自然语言课堂分析报告**

# 五、系统优势分析

---

## 5.1 无需训练复杂行为识别模型

传统方案通常需要：

* 目标检测模型（YOLO）
* 姿态识别模型（Pose）
* 行为分类模型

而本系统直接利用视觉大模型完成行为识别：

> **无需训练检测模型，降低开发难度**

---

## 5.2 系统模块职责清晰

* **Miloco**：负责视频流生成
* **OpenClaw**：负责任务调度与分析
* **视觉模型**：负责行为识别

实现：

> **视频接入、行为识别、统计分析、报告生成全流程闭环**

---

## 5.3 易于扩展

后续可扩展识别行为：

* 交头接耳
* 趴桌
* 离座
* 课堂互动次数

并扩展报告内容：

* 学生专注趋势
* 课堂参与趋势
* 异常行为预警
